üß† How AccessNode Works

AccessNode is composed of four major systems working together:

1. PermissionHandler

Handles the heavy lifting of validating all permission layers:

  • Member and bot permission checks
  • Channel-specific validation
  • Role hierarchy enforcement
  • Moderation action validation
  • Auto-generated error embeds

2. CommandPermissionWrapper

A lightweight wrapper that links permission validation with command execution:

  • Safely runs commands after passing permission checks
  • Automatically reports missing permissions
  • Tracks execution stats

3. CommandMiddleware

Adds automatic permission checking into command flow:

  • Middleware stack for validation and logging
  • Automatic error replies when validation fails

4. CommandExecutor

The final layer that actually runs your command safely:

  • Built-in validation
  • Error handling & recovery
  • Execution history and monitoring
