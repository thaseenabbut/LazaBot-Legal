ActionTag

LazaBot’s unique moderation identifier system that brings structure, clarity, and full traceability to every moderation action.

What is ActionTag?

Every moderation action performed by LazaBot generates a unique identifier called an ActionTag. This tag allows moderators to easily locate, reference, and review the full details of any action — even weeks or months later.

Key Benefits

Unique Identifier

Every moderation action receives a unique ActionTag — no duplicates, always traceable.

Improved Logging

Logs become cleaner, more organized, and easier to filter and search.

Transparency

ActionTags can be shared with users for inquiries or appeals, improving trust.

Professional Moderation

Brings a structured, industry-standard workflow to your moderation team.

ActionTag Format

Each ActionTag follows a structured and consistent format:

ACTION-YYYYMMDD-XXXX

Example:

ACTION-20250117-1234

How ActionTag Works

1. Action Occurs

A moderation command is executed (warn, mute, kick, ban, etc.).

2. Tag Generated

LazaBot automatically generates a timestamp-based ActionTag.

3. Logged in System

The action is stored in LazaBot’s internal logs for later review.

4. View Anytime

Use the command below to retrieve the full action details.

Viewing an ActionTag

Use the command below:

/actiontag <ActionTag>

LazaBot will display a detailed summary including:
• Action type
• User
• Moderator
• Reason
• Timestamp
• Additional metadata
