ActionTag

ActionTag is LazaBot’s unique moderation identifier system designed to make every moderation action traceable, organized, and easy to reference. Whether it's a warning, mute, kick, or ban, each moderation event receives its own ActionTag that allows moderators to quickly locate and review specific actions.

What is ActionTag?

ActionTag is a unique code automatically generated by LazaBot for every moderation event. It serves as a permanent reference ID that can be used to view full details of the action at any time.

ActionTags improve transparency, help during disputes or appeals, and bring a more professional structure to server moderation.

Key Highlights

  • Unique Identifier: Every moderation action receives its own ActionTag.
  • Easy Tracking: Quickly locate and reference actions using their tag.
  • Enhanced Logging: Moderation logs become cleaner and more organized.
  • Transparency: ActionTags can be shared with members for appeals or reviews.
  • Professional Workflow: Standardizes moderation like real-world ticket systems.

ActionTag Format

Each ActionTag is generated using a timestamp and a random ID to ensure uniqueness.

ACTION-YYYYMMDD-XXXX

Example:

ACTION-20250117-1234

Using ActionTag

  • ActionTags are automatically created whenever a moderation command is executed.
  • To view full details of a moderation action, use the command:

    /actiontag <ActionTag>
  • LazaBot will display all available details about that specific action, including:
    • Type of action (Warn, Mute, Kick, Ban)
    • User involved
    • Moderator
    • Timestamp
    • Reason
    • Any additional metadata

Why ActionTag Matters

ActionTag ensures that moderation actions remain reliable and easy to audit. Moderators no longer need to search through logs manually — the ActionTag directs them instantly to the exact action. This dramatically improves the speed, clarity, and fairness of moderation work.
